# 任务指令：修复编辑器富文本格式丢失

## 问题描述
当前编辑器在切换章节时，加粗、颜色、高亮、标题等 TipTap 格式全部丢失，只剩纯文本。

**原因**：父组件（`page.tsx`）在保存和加载章节内容时，只传了纯文本：
- `onChange` 回调里传的是 `editor.getText()`（纯文本）
- 加载时用 `editor.commands.setContent(content)` 恢复，但 `content` 也是纯文本

## 修复目标
切换章节后，此前写入的加粗/颜色/高亮/标题等全部格式不丢失。

## 修复方法

### 改动 1：`writing-editor.tsx` — 改用 HTML 传递内容
```tsx
// 改前（行 40）
onChange(editor.getText())

// 改后
onChange(editor.getHTML())
```

### 改动 2：`page.tsx` — 保存 HTML 而非纯文本
在 `handleSave`、`auto-save`（30s 定时器）、以及手动保存按钮中，存储 `activeChapter.content` 时确保存的是 HTML。

当前手动保存：
```tsx
updateChapterContent(activeChapter.id, content)
```
`content` 是纯文本。需要确认 `store.ts` 中 `updateChapterContent` 的 content 参数类型是 `string`，不用改——只需保证传入的是 HTML。

### 改动 3：`page.tsx` — 章节切换时加载 HTML
在 `setActiveChapterId` 的 `useEffect` 中，加载内容后传给编辑器：
```tsx
// 当前
setContent(ch.content || '')

// 不需要改—setContent 传给 WritingEditor 的 content prop
// WritingEditor 内部用 setContent(content) 恢复
// 只要 content 存的是 HTML，恢复出来就是带格式的
```

### 改动 4：确认 `WritingEditor` 的 `setContent` 接收 HTML
当前 `writing-editor.tsx` 中用 `editor.commands.setContent(content)` 恢复内容。TipTap 的 `setContent` 默认接收 HTML，所以只要 `content` 是 HTML，格式就保留了。

## 关键注意
- **不要**修改 `store.ts` 中的 `updateChapterContent` 函数——它只做字符串存/取，不需要知道存的是纯文本还是 HTML
- **不要**修改 `types.ts` 中的 `Chapter.content` 类型——它已经是 `string`
- 只改 `writing-editor.tsx` 中 `onUpdate` 回调的 `onChange(editor.getHTML())` 这一行

## 涉及文件
| 文件 | 改动 |
|------|------|
| `src/components/writing-editor.tsx` | `onChange(editor.getText())` → `onChange(editor.getHTML())`（行 40 附近） |

## 验收标准
1. 在编辑器中写入一段文字，加粗部分内容，给部分文字加高亮标记
2. 点左侧栏切换到另一章，再点回原来那章
3. 加粗和高亮仍然保留，没有变成纯文本
4. 字数统计、合规检测等功能不受影响
