# 墨境集团 · 搭档互助协议

> 每人一个搭档。卡住出声，搭档补位。

## 搭档对

| 主 | 搭档 | 触发条件 |
|------|------|------|
| 前端 | 页面组件工程师 | 前端 max_steps 3次→搭档接盘 |
| 后端 | API逻辑工程师 | 同上 |
| 编辑器专责 | 前端（反向） | 编辑器卡→前端介入 |
| UX | QA | 走查卡→QA直接测 |
| UI | 动效设计师 | 视觉卡→动效出替代方案 |
| 产品 | 增长分析师 | 需求卡→增长数据兜底 |
| 提示词 | AI评测员 | 提示词卡→评测标准检验 |
| DevOps | 后端（反向） | CI挂→后端看日志 |
| 架构 | 协议执行官 | 审卡→执行官介入调停 |
| 总管 | 组织发展官 | 总管离线→OD暂代调度 |

## 规则
1. 卡住的人必须在 `context/inbox/搭档名.jsonl` 写求助消息
2. 搭档收到后 1 轮内回应（要么帮忙要么升级到总管）
3. 搭档解决不了→总管介入
4. 同一人 3 次求助解决不了→组织发展官评估是否换岗

## 求助消息格式
```json
{"from":"角色","type":"help","stuck_on":"什么事卡住了","tried":"已尝试的方法","need":"需要什么帮助"}
```
