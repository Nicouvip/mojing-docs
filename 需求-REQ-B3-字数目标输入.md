# 需求文档：字数目标手动输入

## 元信息
- **需求编号：** REQ-B3
- **优先级：** P1 (Should)
- **RICE 评分：** 12/100
- **涉及角色：** 前端技术负责人
- **预估工作量：** 0.3 人天
- **依赖：** 无

---

## 用户故事

> **As a** 每日有码字目标的作者
> **I want** 能手动输入我今天的写作目标字数，而不是只能点 `+500`
> **So that** 我可以设任意目标（如 2000、4000），更好地规划每日写作量

**现状问题：** 字数目标处仅有 `+500` 按钮，每点一次增加 500 字。用户无法直接输入自己想要的目标字数（如 3000、6666），每次都要狂点按钮。

---

## 功能拆解

### 3.1 UI 改动
在当前字数目标显示区域（编辑区底部），增加编辑入口：

**方式一（推荐）：点击目标数字进入编辑模式**
- 当前显示："目标: 2000字"
- 点击后变为可编辑输入框（`<input type="number">`）
- 输入数字后回车确认、失焦确认、Esc 取消
- 输入框自动获得焦点

**方式二：目标数字旁加一个 ✏️ 小图标**
- 点击 ✏️ 弹出小输入框
- 输入后确认更新

### 3.2 状态存储
文件：`src/lib/store.ts`

- 当前目标字数存储在 `projects` 的 `writingGoal` 字段（或 `chapter?.writingGoal`）
- 需要确保手动输入的值持久化到 localStorage
- 目标字数应在章节切换后保持（和项目关联，而非单个章节）

### 3.3 输入验证
| 条件 | 行为 |
|------|------|
| 输入 0 或负数 | 视为"不设目标"，进度条隐藏或显示"未设定目标" |
| 输入超大数字（>10万） | 提示"目标字数过大"但允许设置（为用户自由留空间） |
| 非数字输入 | 输入框类型为 `number`，默认不允许非数字 |
| 空输入 | 恢复为上次有效值 |

---

## 验收标准

1. ✅ 点击当前目标字数，变为可编辑输入框
2. ✅ 输入新数字后回车确认，目标更新、进度条重新计算
3. ✅ Esc 取消编辑，恢复原值
4. ✅ 刷新页面后，手动设定的目标不丢失
5. ✅ `+500` 按钮仍在，修改目标后点 `+500` 从新目标基础上增加
6. ✅ 输入 0 或负数，进度条区域显示"未设定目标"

---

## 边界条件

- 进度条当前百分比 = `当前字数 / 目标字数 × 100%`
- 如果目标字数设为 0，进度条不应显示 Infinity%
- 切换章节时，目标字数保持（按项目设定）

---

## 团队备注

- 改动范围小：仅在 page.tsx 中修改目标字数显示区域的 UI
- 如当前 `writingGoal` 只存在内存中未持久化，需补充 localStorage 读写
- 目标字数是按项目设定还是按章节设定？建议按项目（一次性设好，所有章节共享同一个目标）
