# 墨境项目版本演变史

从三个备份对比还原出的完整演变脉络：

## V1 — 原始版（Coze/扣子 AI 生成）
**来源**: 最新的备份 tar（有 .cozeproj 目录的那一个）
- **架构**: 多页面 — `/`、`/novel`、`/editor`、`/tools`、`/works` 各自独立
- **设计**: 蛙蛙风格 — 薄荷绿 #10B981（与 .cozeproj HTML 原型一致）
- **特点**: 有 `src/app/novel/page.tsx`（398行）、`src/app/tools/page.tsx`、`src/app/works/page.tsx`（598行）
- **组件**: home-page.tsx, writing-app.tsx, onboarding-guide.tsx 等
- **UI库**: 更完整的 shadcn/ui（有 accordion, breadcrumb, carousel, chart, calendar 等）
- **有** admin 后台页面（dashboard, users, settings, prompts 等12个页面）
- **有** database/init.sql + import-backup-data.sql
- 色系是蛙蛙的薄荷绿

## V2 — 改造版（手动迭代）
**来源**: `mojing-full-backup-20260615-192924`（含 components-v2/ 和 v2/）
- **架构**: 单页 SPA — `page.tsx` → `WritingApp` 一个页面撑起整个应用
- **设计**: 改为苹果蓝 #0071E3 + 紫渐变（去掉了蛙蛙的薄荷绿）
- **新增**: brain-jet-wizard, storm-wizard, incubate-wizard, title-wizard 四步流水线向导
- **新增**: compliance-badge, compliance-actions（合规检测 UI）
- **新增**: cooling-panel（技法冷却面板）
- **新增**: diff-viewer, material-visual-panel
- **新增**: 偏好学习系统（lib/preferences.ts）
- **数据库**: Drizzle ORM schema 22张表
- **混乱源**: 有 components-v2/, src/v2/, page.tsx.bak 等残留

## V3 — 精简版（只打包了后端）
**来源**: mojing-backend.tar.gz（用户给的第一个文件）
- 只有 `src/app/api/` + `src/lib/` + `src/hooks/` + `src/storage/`
- 没有前端页面
- server/ 目录包含 Express 后端 + JSON 数据

## 数据库状态（Supabase）
- 1个用户: nicou@163.com（admin）
- 站点名: "墨境小说创作站"
- 配置: 免费项目上限3个，每日AI调用10次，开放注册，未维护模式
- 有 operation_logs（历史操作记录）

## 用户的核心诉求
"越改越多错误" = V1→V2 的改造过程中残留文件导致架构冲突，编译出错。

**Why:** 了解版本演变才能理解为什么"越改越乱"，从而给出针对性方案。
**How to apply:** 后续处理时，需要先确定走 V1 的多页架构还是 V2 的 SPA 架构，清理残留，统一色系。
