## 二、测试用例

### TC-01：创建新作品

| 项目 | 内容 |
|------|------|
| **描述** | 用户新建一个小说作品 |
| **前置条件** | 在首页 |
| **步骤** | ① 点击"新建作品"按钮 → ② 输入作品名称 → ③ 选择题材 → ④ 点击"创建" |
| **预期结果** | 自动跳转到编辑器页，左侧章节树出现"第一章"，字数统计为 0 |
| **验证要点** | 路由跳转 `/editor/[id]`；章节默认创建；localStorage 数据写入 |

### TC-02：作品列表展示

| 项目 | 内容 |
|------|------|
| **描述** | 首页展示所有已创建作品 |
| **前置条件** | 已创建 ≥1 个作品 |
| **步骤** | 首页加载 |
| **预期结果** | 展示作品卡片：名称、题材标签、章节数、字数、"继续创作"按钮 |
| **验证要点** | 从 localStorage 读取数据正常渲染 |

### TC-03：删除作品

| 项目 | 内容 |
|------|------|
| **描述** | 用户删除一个作品及其全部章节 |
| **前置条件** | 已创建 ≥1 个作品 |
| **步骤** | ① 鼠标悬停作品 → ② 点击删除图标（垃圾桶） |
| **预期结果** | 作品从列表消失，对应章节全部从 localStorage 删除 |
| **边界** | 删除后确认不再出现；章节数据不可恢复 |

### TC-04：富文本编辑 — 加粗/斜体/下划线/高亮

|------|------|
| **描述** | 使用工具栏对选中文字应用格式 |

| **步骤** | ① 选中文字 → ② 分别点击 B/I/U/🖍 按钮 |
| **预期结果** | 文字被正确应用对应格式，按钮高亮显示激活状态 |
| **验证要点** | 格式可叠加；取消选中后格式仍然保持 |

### TC-19：合规检测 — 实时检测

| 项目 | 内容 |
|------|------|
| **描述** | 编辑时实时检测违规并显示 |
| **前置条件** | 编辑器有内容 |
| **步骤** | ① 输入含 B 类禁用词（如"突然"重复≥3次）的段落 → ② 按 Enter |
| **预期结果** | 右下角出现违规计数浮窗；右侧合规面板显示违规详情 |

### TC-20：合规检测 — 六大规则

| 编号 | 规则 | 检测方式 | 测试方法 |
|------|------|----------|----------|
| 1 | B 类禁用词（同段≥3次） | forbiddenB | 输入含 3+ "突然/感觉/非常"的段落 |
| 2 | A 类递进判断句 | forbiddenA | 输入"不是…而是…"句式 |
| 3 | C 类连接词超限 | forbiddenC | 输入多个"然后/于是/就" |
| 4 | 动作句后跟解释 | explanation_after_action | 输入"他站起身，因为他很生气" |
| 5 | 55字生死线 | check55CharLine | 段落前55字无冲突/悬念词 |
| 6 | 身体密度 | calcBodyDensity | 检测含身体动作句的比例 |
