# 知识库文件自动导出指令 V1.1

## 触发指令

作者输入 `/导出知识库`，AI 自动执行本指令。

## 执行规则

AI 从当前对话上下文或作者提供的原料包中，提取所有可用信息，按以下蓝图生成 7 个知识库文件。**不需要原料包包含特定字段**——AI 从已有信息中提取和推断，缺失字段自动标注 `[未提供]` 或 `[AI推断]`。

## 输出规范

每个知识库文件用独立的 Markdown 代码块包裹，代码块前用标题标注文件名。作者可直接复制每个代码块内容，保存为 `.txt` 文件，上传至 Cherry Studio 知识库。


## 文件一：人物设定卡合集

**文件名**：`知识库_人物设定卡合集.txt`

**提取规则**：从原料包或对话历史中提取所有已出场/已设定角色，每人一张设定卡。

**格式**：

```
## [角色名]

- **身份**：[提取或推断]
- **性格关键词**：[提取或推断·≤5个词]
- **身体习惯**：[紧张/说谎/心动/思考时的下意识动作。若未提供，AI根据性格推断并标注[AI推断]]
- **说话风格**：[语速/常用词/口头禅/句子长度。若未提供，标注[未提供]]
- **描写指纹**：[感官通道+专属意象或比喻域。若未提供，AI推荐并标注[AI推荐]]
- **核心欲望**：[提取或推断]
- **核心恐惧**：[提取或推断]
- **与主角关系**：[提取或推断·包含表面关系与隐藏张力]
```

## 文件二：人物关系网络

**文件名**：`知识库_人物关系网络.txt`

**提取规则**：从角色档案、大纲、正文中提取所有主要人物之间的关系。

**格式**：

```
## [角色A] — [角色B]

- **表面关系**：[日常互动层面]
- **隐藏张力**：[底层未言明的关系暗流·若未提供，AI根据情节推断并标注[AI推断]]
- **关键交集**：[两人在情节中的核心交汇点]
```

---

## 文件三：世界设定与编号体系

**文件名**：`知识库_世界设定与编号体系.txt`

**提取规则**：从原料包世界观快照、大纲、正文中提取所有设定信息。

**格式**：

```
## 一、故事背景
- **时代与地点**：[提取或推断]
- **世界本质**：[表面与底层的双层描述]

## 二、核心设定
[逐条列出所有已知规则/体系/机制，每条标注来源，推断项标注[AI推断]]

## 三、组织与势力
[逐个列出所有已知组织，包含定位、核心人物、运作规则]

## 四、编号/命名体系
[列出所有已知编号前缀及其含义，附已知编号清单]

## 五、关键地点
[逐个列出，包含位置、首次出现章、与主线的关联]

## 六、关键术语
| 术语 | 释义 |
|:---|:---|
|[提取所有专有名词及其解释]|
```

---

## 文件四：时间线与大事件编年史

**文件名**：`知识库_时间线与大事件编年史.txt`

**提取规则**：从大纲、正文中提取所有有明确时间节点的事件，按时间排序。

**格式**：

```
## 故事前史（故事开始前）
- **[时间点]**：[事件] [来源]

## 第一卷
- **第X章**：[核心事件·≤30字] [功能标签] [情感走向]
...

## 第二卷
...
```

若原料包无明确年份，以故事当前时间点为基准，用相对时间标注（如“六年前”“三个月前”）。

---

## 文件五：专属意象与道具词典

**文件名**：`知识库_专属意象与道具词典.txt`

**提取规则**：从角色描写指纹、正文中提取在两个及以上章节中出现的相同意象、道具、标志性物品。

**格式**：

```
## [意象/道具名]

- **关联角色**：[角色名]
- **首次出现**：[章节]
- **重复出现**：[章节列表]
- **象征意义**：[AI根据全书情感底色和角色弧推断·标注[AI推断]]
```

若全书尚未写够两章，列出已出现的潜在重复意象并标注 `[待确认是否重复使用]`。

---

## 文件六：风格规则速查卡

**文件名**：`知识库_风格规则速查卡.txt`

**提取规则**：从原料包创作宪章、对话历史中的模板规则提取。

**格式**：

```
## 总体规则
- **语言风格**：[提取·冷峻白描/快消口语/感官极值]
- **目标读者**：[提取或推断]
- **情感底色**：[提取]

## 写作硬约束
- **身体优先**：情绪必须通过身体反应承载
- **单一意图**：一个语义段落只做一件事
- **三不原则**：不定义情绪、不解释动机、不总结道理
- **评价性锚点**：[提取密度要求·快消口语适用]
- **对话占比**：[提取目标值]

## 核心技法
[逐条列出]

## 圈层痛点
- **普适痛点**：[提取]
- **圈层痛点**：[提取]
- **深层痛点**：[提取]
```

若原料包未提供完整创作宪章，AI 根据故事类型和题材自动推荐默认参数，并标注 `[AI推荐·建议作者确认]`。

---

## 文件七：前情梗概（已完成章节一句话索引）

**文件名**：`知识库_前情梗概.txt`

**提取规则**：从大纲或已写正文中提取每章核心事件，浓缩为一句话。

**格式**：

```
## 第一卷

| 章 | 标题 | 一句话梗概 |
|:---|:---|:---|
| 1 | [标题] | [≤30字] |
| 2 | [标题] | [≤30字] |
...
```

未写章节不列入。

```
4. 知识库_时间线与大事件编年史.txt
5. 知识库_专属意象与道具词典.txt

📌 以上文件可直接上传至 Cherry Studio 知识库。建议每写完 10 章重新执行一次 `/导出知识库` 以更新前情梗概和时间线。
```


## 使用方式

1. 在包含本指令和原料包的对话中，输入 `/导出知识库`。
2. AI 自动根据原料包内容生成全部 7 个文件。
3. 每个文件以独立 Markdown 代码块输出，直接复制保存为 `.txt`。
4. 上传至知识库，即可用于辅助新章节创作。
