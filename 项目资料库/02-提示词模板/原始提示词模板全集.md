# 墨境的源头：全套原始提示词模板

用户原本是在 AI 对话框中用模板和知识库进行小说创作，所有的规则、技法、流程都是在这套体系中沉淀出来的。墨境网站的目标就是把这些模板可视化成可操作的网站。

## 创作工具链（5个工具，按使用流程排列）

### 1. 脑洞喷射 V1.5
- **定位**: 零门槛高频脑洞生成。输入关键词或无输入，输出5-10个故事脑洞
- **模式**: A完全随机/ B关键词/ C组合裂变/ D约束喷射
- **素材池**: 5大静态永久池(主角身份/核心冲突/情感底色/世界观/开篇场景) + 联网增量池
- **题材适配表**: 10种题材分别有优先身份/冲突/世界观/场景池 + 禁止生成项
- **深化包**: 选定脑洞后可生成标准就绪块，对接灵感爆裂

### 2. 灵感爆裂 V2.1
- **定位**: 全功能创意发散引擎。输入碎片输出结构化故事种子
- **模式一:三问收束** — 模糊碎片时问3个问题收束方向
- **模式二:直接发散** — 4步流程:方向裂变(10个)→走向深化→开篇画面(3个)→输出就绪块
- **模式三:精炼提炼** — 大段设定时提取核心规则
- **模式四:范文仿写** — 拆解范文→提取骨架→变量裂变→5个新方向
- **辅助功能**: 爆款雷达(联网)/拆书/对标拆解/快速仿写
- **核心规则**: 分步确认、用户主导、不代写正文、温度参数提醒、联网标注来源

### 3. 灵感引擎 V5.2
- **定位**: 创意孵化器。接收就绪块，输出标准化原料包(可直接入库)
- **模块一:元素引爆台** — 变量拆解+多版本生成+A/B/C版选择
- **模块二:剧情风暴** — 剧情树推演+变奏生成+因果链检查+情节功能分配+大纲框架
- **模块三:人设深渊镜** — 角色定义+深层挖掘+对接V8.5.0字段
- **模块四:世界观孵化器**(可选) — 17个核心问题+咬合校验
- **模块五:输出原料包** — 标准化V5.2格式，含8大部分
- 支持静默孵化模式(跳步直达)

### 4. 书名炼金术 V1.7
- **定位**: 书名策划工具，基于原料包生成3-5个黄金书名
- **5步流程**: 智能分流→硬约束构造→批量生成(16+候选)→主编筛选→输出入库指令
- **四大书名公式**: 画面型/悬念型/冲突型/精炼型
- **10种题材书名风格**: 悬疑冷峻/都市接地气/玄幻宏大/言情温柔等
- **联网查重**: 验证是否与已有作品重名

### 5. 小墨创作助手 V10.0.2（核心写作引擎）
- **定位**: 实际小说章节写作引擎，分阶段工作流
- **阶段零:启动与规划** — 预选技法+生成A-8状态行
- **阶段一:正文写作** — 只带核心规则+预选技法创作
- **阶段二:写作中快速自查** — 每段后8项检查(含实时阻断)
- **阶段三:章末交付与自检** — 23项检查+润色+冷却更新+状态输出
- **核心规则体系**:
  - A-1全域铁律(最高): 单一意图/身体优先/不可替代瞬间/视角一致性/角色一致性/时间线一致性 + 三不原则(不定义情绪/不解释动机/不总结道理)
  - A-1.1防守规则(与A-1同级): 动作后禁解释/精致句分配硬约束/禁止生造比喻/示例防御
  - A-2禁用词: A类递进判断句/B类常规禁用词/C类连接词/D类上下文/AI高频词汇五类
  - A-3三层红线: L1密度超限→章末替换/L2跳过身体→实时标注/L3情绪无升级→标注
  - 55字生死线(强制): 开篇55字内出现冲突/悬念/动作
  - 冲突强度分级L1-L5: 句子节奏/句号换行/身体密度/形容词密度递进变化
- **三种风格**: 冷峻白描(默认)/快消口语/感官极值
- **三种模式**: 标准/精修/极简

## 与墨境的对应关系

| 原始模板 | 墨境系统中的对应 |
|---------|----------------|
| A-1全域铁律 | L1 铁律层 |
| 灵感引擎模块二(剧情风暴) | L2 项目上下文 |
| 技法库(S1-S6/E1-E12/H01-H13) | L3 技法选择器 + 冷却系统 |
| 文风画像/角色语料库/用户偏好 | L4 偏好学习 |
| 四级净化体系 | 合规检测引擎(8项规则) |
| 冷却系统 | technique_usage表 + cooling-panel |
| 原料包/就绪块 | materials表 |
| 全局摘要协议 | global_summaries表 |
| 脑洞喷射→灵感爆裂→灵感引擎→书名炼金术 | 四步创作流水线 |
| 小墨创作助手阶段一二三 | 正文写作生成流程 |

**Why**: 这是墨境产品最早的"产品需求文档"——用户通过大模型对话验证了所有规则的有效性，现在要把这些规则从手动粘贴变成可视化的网站系统。
**How to apply**: 在设计墨境的每个功能模块时，回到这些原始模板看"用户原本是怎么手动操作的"，网站就是把每一步操作变成按钮和界面。
