# 层层派活协议

## 核心原则

### 1. 组织架构：四级递进链

**总管 → 副手 → 组长 → 一线**

形成自上而下的四层管理链路，每一层只对下一层负责，层层递进。

### 2. 职责分离：每层只派活不干活

每一层的核心职责是**派发任务、追踪进度、问题过滤、人员调度**，而非亲自动手执行。执行工作由最底层（一线）完成，上层专注于管理和协调。

### 3. 问题过滤：层层过滤，逐级上报

问题在每一层被识别、判断、解决或过滤。能在本层解决的不往上传，解决不了的汇总、提炼后上报给上一层。**最高层只收大事**——只有经过多层过滤仍无法解决的重大问题才会到达总管。

### 4. 自动换人：卡住即换，无需请示

当某一任务在执行层卡住（进度停滞、无法推进）时，**当前层自动更换执行人**，无需向上级请示。这保证了系统的自愈能力和响应速度，避免因等待审批而浪费整体时间。

### 5. 能力对等：每层能力等于上一层

每一层的能力模型与上一层保持一致，同样具备四项核心能力：

| 能力 | 说明 |
|------|------|
| **派发** | 准确判断任务性质，分派给合适的下一层或执行人 |
| **追踪** | 持续跟进任务进度，掌握关键节点 |
| **换人** | 在卡顿时果断替换执行人，不依赖上级决策 |
| **汇报** | 向上层提炼、总结、汇报关键信息 |

每层的能力集相同，但视野范围和决策边界不同——下层聚焦具体任务，上层俯瞰全局方向。
