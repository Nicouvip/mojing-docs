## 一、项目名

**墨境 (Mojing)** — AI 网文写作工作台。

将原本需要手动粘贴的 225KB 提示词规则变成可视化的网站操作，覆盖灵感发散、书名策划、脑洞生成、创作续写、润色精修全流程。

---

## 二、技术栈

| 层 | 技术 | 用途 |
|---|------|------|
| **前端** | Next.js 16 + React 19 + TypeScript | 主站框架 |
| **样式** | Tailwind CSS 4 + shadcn | 原子化 CSS + 组件库 |
| **编辑器** | TipTap 3 + GrapesJS | 富文本编辑器 + 页面拖拽构建 |
| **图标** | lucide-react | 统一图标系统 |
| **状态** | React Context + localStorage → Supabase 迁移中 | 数据层 |
| **数据库** | PostgreSQL 15 + Drizzle ORM | 持久化 / ORM |
| **认证** | NextAuth.js 5 beta + bcryptjs | 登录注册 |
| **AI** | 自建提示词系统（5工具 + 7知识库, 225KB） | AI 写作能力 |
| **后端** | Supabase 自托管（GoTrue / PostgREST / Realtime / Storage） | 认证 / REST API / 实时 / 文件 |
| **网关** | Kong 3.8 (Docker) | API 路由 & 限流 |
| **包管理** | pnpm | 依赖管理 |
| **测试** | Playwright | 自动化截图测试 |
| **部署** | Docker Compose | 基础设施编排 |

## 四、如何启动（3 个端口）

### ① 主站（Next.js 开发服务器）
```bash
cd D:\建网站\mojing-app
pnpm dev
```
→ **http://localhost:3000**

| 路径 | 说明 |
|------|------|
| `/` | 首页（文学杂志感 Landing Page） |
| `/editor/demo-1` | 编辑器 |
| `/admin` | 后台管理 |
| `/login` / `/register` | 登录注册 |
| `/dashboard` / `/works` / `/account` | 用户仪表盘 / 作品 / 个人中心 |

### ② 项目全景面板（团队监控）
```bash
# 由 mojing-tools/server.py 提供
python D:\建网站\mojing-tools\server.py
# 或双击 墨境悬浮窗.vbs
```
→ **http://localhost:8899**

实时团队状态、服务器监控、进度仪表盘。

### ③ Supabase Studio（数据库管理面板）
```bash
cd D:\建网站\mojing-infra
docker compose up -d
```
→ **http://localhost:8082**

| 服务 | 端口 | 说明 |
|------|:----:|------|
| Supabase Studio | 8082 | Web 数据库管理 |
| Kong API 网关 | 8000 | API 路由入口 |
| PostgreSQL | 5432 | 数据库 |
| Realtime | 4000 | WebSocket 实时服务 |

> 其他端口：会议室服务 → `http://localhost:8080`（Go WebSocket 聊天）。

## 五、5 组职责（C-G）

### C组 — 工具组（组长 + 4人）
- **强制技能**：`frontend-design`
- **职责**：`mojing-tools/server.py` 升级、WebSocket 服务、统一仪表盘、游戏界面 React 化
- **产出目标**：工具服务端 + 前端工具面板

### D组 — 基础组（组长 + 4人）
- **强制技能**：`executing-plans`
- **职责**：Supabase 真接入、Drizzle ORM 建表、数据迁移脚本、CI/CD 生产化
- **产出目标**：数据库层 + 迁移管线 + 部署流水线

### E组 — 测试组（组长 + 5人）
- **强制技能**：`webapp-testing` + `systematic-debugging`
- **职责**：自动化测试、性能测试、安全测试、兼容性测试、回归测试
- **工作方式**：紧跟 A/B/C/D 四组，每组产出后 24 小时内出测试报告

### F组 — 审查组（组长 + 3人）
- **强制技能**：`flow-review` + `chinese-code-review`
- **职责**：逐 PR 代码审查、架构一致性检查、安全审查
- **门禁规则**：每次代码合并前必须通过 F 组审查

### G组 — 文档组 🧑‍💼（组长 + 3人）
- **强制技能**：`chinese-documentation`
- **职责**：API 文档、用户手册、开发者指南、变更日志
