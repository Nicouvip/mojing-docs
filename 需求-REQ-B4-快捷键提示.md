- **RICE 评分：** 9/100
- **涉及角色：** 前端技术负责人
- **预估工作量：** 0.3 人天
- **依赖：** 无

---

## 用户故事

> **As a** 正在全屏专注写作的作者
> **I want** 鼠标悬停在工具栏按钮上时，能看到对应的快捷键提示
> **So that** 我可以逐步记住快捷键，减少鼠标操作，提升写作效率

**现状问题：** 编辑器工具栏各按钮（加粗、斜体、标题等）均无快捷键标注。新用户不知道 Ctrl+B 是加粗、Ctrl+I 是斜体，每个操作都要点按钮，打断写作流。

---

## 功能拆解

### 4.1 工具栏按钮加上 title 属性
文件：`src/components/writing-editor.tsx`

为每个工具栏按钮添加 `title` 属性，鼠标悬停时显示功能名 + 快捷键：

| 按钮 | title 文本 |
|------|-----------|
| 加粗 **B** | `加粗 (Ctrl+B)` |
| 斜体 *I* | `斜体 (Ctrl+I)` |
| 下划线 U | `下划线 (Ctrl+U)` |
| 标题 H1-H3 | `标题 1 (Ctrl+Alt+1)` / `标题 2 (Ctrl+Alt+2)` / `标题 3 (Ctrl+Alt+3)` |
| 引用 | `引用 (Ctrl+Shift+B)` |
| 无序列表 | `无序列表 (Ctrl+Shift+8)` |
| 有序列表 | `有序列表 (Ctrl+Shift+7)` |
| 分隔线 | `分隔线` |
| 左对齐 | `左对齐` |
| 居中对齐 | `居中对齐` |
| 右对齐 | `右对齐` |
| 撤销 ↩️ | `撤销 (Ctrl+Z)` |
| 重做 ↪️ | `重做 (Ctrl+Y)` |
| 高亮标记（颜色） | `标记颜色` / 具体颜色名 |

### 4.2 实现方式
- 直接使用 HTML `title` 属性，不需要额外组件
- 已在当前 `toolbarButtons` 数组中的每个 button 元素上加 `title` prop

### 4.3 快捷键一致性
- 与 TipTap 默认快捷键保持一致
- 如果自定义了快捷键，在 title 中优先显示自定义的组合

---

## 验收标准

1. ✅ 鼠标悬停在加粗/斜体/下划线按钮上，显示对应的快捷键提示
2. ✅ 鼠标悬停在 H1/H2/H3 标题按钮上，显示快捷键提示
3. ✅ 鼠标悬停在引用、列表按钮上，显示快捷键提示
4. ✅ 鼠标悬停在撤销/重做按钮上，显示快捷键提示
5. ✅ 快捷键提示格式统一为 `"功能名 (快捷键)"`

---

## 边界条件

- Mac 用户：显示 `⌘B` 还是 `Ctrl+B`？TipTap 默认支持 Ctrl（Windows）和 ⌘（Mac），title 统一显示为 `Ctrl+B` 最简单，用户自己对应平台
- 如果后续加自定义快捷键，title 应同步更新

---

## 团队备注

- 最简单的改动：在现有按钮的 `<button>` 元素上加 `title` 属性
- 不需要新 UI 组件，不需要工具提示（Tooltip）组件
- 可一步到位，不需要分步实施
