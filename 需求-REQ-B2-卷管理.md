# 需求文档：卷管理（重命名/删除/章节分配）

## 元信息
- **需求编号：** REQ-B2
- **优先级：** P1 (Should)
- **RICE 评分：** 9/100
- **涉及角色：** 前端技术负责人
- **预估工作量：** 1.5 人天
- **依赖：** 数据层可能需要扩展

---

## 用户故事

> **As a** 写长篇小说的作者
> **I want** 能重命名卷、删除空卷、自由分配章节到不同卷
> **So that** 我的长篇作品可以有清晰的卷结构，像实体书一样组织

**现状问题：** 当前卷管理功能严重残缺：
1. 卷右侧"···"按钮点击仅弹出 alert('卷功能待实现')
2. 章节并非用户分配到卷，而是按 `Math.ceil(displayChs.length / volumes.length)` 等分
3. 无法重命名卷、删除卷、将章节从一个卷移到另一个卷

---

## 功能拆解

### 2.1 卷数据结构扩展
文件：`src/lib/types.ts`

当前卷定义较简单（如 `{ id, name, order }`），需要扩展为：
```typescript
interface Volume {
  id: string;
  projectId: string;
  name: string;
  order: number;        // 排序序号
  createdAt: number;
  updatedAt: number;
}
```

并为 Chapter 增加 `volumeId` 字段：
```typescript
interface Chapter {
  // ...现有字段
  volumeId: string;     // 所属卷 ID
  // ...其他字段
}
```

### 2.2 卷存储操作
文件：`src/lib/store.ts`

新增卷操作函数：
| 操作 | 函数 | 说明 |
|------|------|------|
| 读取所有卷 | `getVolumes(projectId)` | 按 order 排序返回 |
| 创建卷 | `createVolume(projectId, name)` | 自动生成 id 和 order |
| 重命名卷 | `renameVolume(volumeId, name)` | 更新 name |
| 删除卷 | `deleteVolume(volumeId)` | 仅允许删除空卷（无章节） |
| 调整卷顺序 | `reorderVolumes(projectId, volumeIds)` | 批量更新 order |

### 2.3 卷 ··· 菜单实现
当前卷右侧的"···"按钮弹出 `alert('卷功能待实现')`，改为弹出下拉菜单：

| 菜单项 | 操作 |
|--------|------|
| ✏️ 重命名 | 弹出内联输入框（或小弹窗），输入新名称后确认 |
| ❌ 删除卷 | 先检查卷内是否有章节 → 空卷直接删除；非空卷提示 "卷内有 X 篇章节，删除后章节将移入未分类" |
| ➕ 新建章 | 在此卷下新建章节（已有功能） |

### 2.4 章节拖拽到不同卷（可选，P2）
如时间充裕，实现章节在卷之间拖拽移动：
- 左侧栏章节列表中，长按或拖拽章节到其他卷区域
- 同时更新 chapter.volumeId

### 2.5 默认卷
- 每个项目至少有 1 个默认卷（"第一卷"）
- 删除项目所有卷时自动创建默认卷
- 新创建章节默认归入第一个卷

---

## 验收标准

1. ✅ 点击卷的"···"按钮，弹出菜单含"✏️ 重命名"和"❌ 删除卷"
2. ✅ 重命名时，内联输入框出现在卷名位置，回车确认、Esc 取消
3. ✅ 删除空卷时，卷直接从列表中消失
4. ✅ 删除有关章节的卷时，提示用户章节将移入未分类
5. ✅ 新建章节时，默认归入第一个卷
6. ✅ 已有数据的项目，旧章节自动分配到默认卷（向前兼容）

---

## 边界条件

- 至少保留 1 个卷：如果只剩最后一个卷，不允许删除
- 重命名不能为空名或纯空格：输入空名时恢复原名
- 卷名长度限制：不超过 50 个字符
- 旧数据兼容：现有项目没有 `volumeId` 的章节，视为归入"未分类"或默认第一卷

---

## 团队备注

- 本改动涉及 types.ts（类型扩展）、store.ts（新增操作）、page.tsx（UI 交互）
- 如果改造成本过高，考虑分步实施：先做重命名和删除，章节分配放到下一轮
- 建议使用 `alert` 或 shadcn 的 Dialog 组件做删除确认
