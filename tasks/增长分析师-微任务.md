**微任务：首个埋点采集 + 留存看板 SQL**

1. 在项目 `src/lib/analytics/` 下新建 `events.ts`，定义 3 个核心埋点事件：
   - `page_view`（页面浏览，带 path / referrer / timestamp）
   - `story_create`（用户点击"开始创作"，记录耗时秒数）
   - `story_publish`（成功发布，记录字数 / 章节数）
2. 创建 SQL 文件 `analytics/retention-dashboard.sql`，写出可查「昨日 DAU / 7 日留存率 / 完稿率（发布÷创建）」的查询
3. 输出到 `output/增长-埋点与看板方案.md`

验收标准：`events.ts` 类型安全、无其他依赖；SQL 查询在 PostgreSQL 15+ 中可直接运行并返回正确聚合。
