【强制技能 — flow-review + chinese-code-review】
────────────────────────

**微任务：制定 PR 审查流程 + 首次代码审查模板**

1. 在项目根目录创建 `.github/PULL_REQUEST_TEMPLATE.md`，包含以下区块：
   ```markdown
   ## 描述
   <!-- 该 PR 解决了什么问题 -->

   ## 变更类型
   - [ ] 功能新增
   - [ ] Bug 修复
   - [ ] 重构
   - [ ] 文档
   - [ ] 其他

   ## 自检清单
   - [ ] 已添加/更新测试
   - [ ] 已通过 `pytest`（后端）/ `npm test`（前端）
   - [ ] 无新引入的 lint 警告
   - [ ] 已更新相关文档

   ## 截图（UI 变更时必填）

   ## 关联 Issue
   ```
2. 创建 `docs/review-checklist.md`，列出 F 组审查的必查项（≥10 项），涵盖：
   - 代码风格一致性 · 错误处理完整性 · 安全漏洞（SQL 注入/XSS） · 性能隐患 · 日志规范 · 测试覆盖 · 数据库迁移兼容性 · API 兼容性 · 配置可管理性 · 文档同步
3. 在根目录创建 `scripts/review.sh`（或 `review.ps1`）辅助脚本：
   - 拉取目标分支 → 列出待审查文件 → 输出差异统计
4. 输出到 `output/F组-审查流程与模板.md`

验收标准：新建 PR 时自动加载模板；审查清单可供审查员逐项勾选；辅助脚本可列出待审查文件清单。
