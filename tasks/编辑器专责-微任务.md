**微任务：编辑器自动保存防丢机制 —— 草稿本地持久化**

1. 在 `src/app/editor/[id]/` 下实现 `useAutoSave` hook：
   - 内容变更后 **3s 防抖**写入 `localStorage`，key 为 `draft-${id}`
   - 每次写入记录时间戳，限制单篇草稿大小不超过 500KB（超出时提示"内容过长，部分未保存"）
2. 编辑器初始化时检查 `localStorage` 有无过期草稿（>24h 视为过期自动清理），若有比服务端更新的草稿，弹出恢复确认对话框
3. 开启页面关闭监听（`beforeunload`）：如果 3s 内有过未保存变更，提示用户
4. 在 `src/lib/editor-draft.ts` 中抽取通用逻辑，写单元测试覆盖：防抖时序、过期清理、大小超限
5. 输出到 `output/编辑器-自动保存防丢方案.md`

验收标准：编辑器输入内容后 3s 关闭页面 → 再打开看到恢复弹窗 → 确认后内容还原；F12 Application → Local Storage 可看到 `draft-*` 记录；单元测试通过。
