**微任务：API 响应性能审计与优化**

1. 遍历 `src/app/api/` 下所有路由，输出每条的响应时间中位数（P50）和 P95
2. 识别响应 > 500ms 的端点，标记瓶颈原因（数据库查询 / AI 调用 / 无缓存）
3. 对 Top-3 慢端点各写一个优化方案并实施（加索引 / 加 Redis 缓存 / 异步化）
4. 输出到 `output/后端-API性能审计报告.md`

验收标准：优化后 P95 ≤ 300ms，提供优化前后的对比数据。
