**微任务：页面级骨架屏（Skeleton Loading）补全**

1. 遍历 `src/app/` 下所有 15 个页面路由，标记每个页面当前加载态的表现（无加载态 / 用 Spinner / 用骨架屏 / 完美）
2. 对 **无加载态** 和 **用 Spinner** 的页面，实现 Tailwind + CSS 骨架屏：
   - 每个页面创建一个 `Skeleton.tsx` 组件（放在该页面目录下），模拟页面真实布局的占位灰块（标题行、卡片、表格行）
   - 使用 `animate-pulse` Tailwind 类，无需额外 JS 依赖
3. 在父组件中用 `Suspense` 包裹，fallback 指向骨架屏组件
4. 输出到 `output/页面-骨架屏补全报告.md`（附 Before/After 截图描述）

验收标准：打开任何一个原本空白/Spinner 的页面，首次加载时看到与最终内容轮廓一致、带呼吸动效的灰色骨架块；网络慢 3G 时骨架屏先于内容出现；无新 npm 依赖。
