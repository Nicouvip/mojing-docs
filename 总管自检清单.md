# 总管自检清单

> 每次收工前、每次说"看一下进度"后，必须逐条检查。漏一条 = 失职。

## 进度检查
- [ ] 所有 8 个角色状态（🟢活跃/⏸空闲/❌报错）
- [ ] 最新产出文件是否已写入 output/
- [ ] 是否有角色卡住（最后一条是空回复或"正在思考"）
- [ ] 任务文件是否需要更新（分配新任务/修正方向）

## 文档同步
- [ ] 决策日志：本轮有重大决策？立刻补录
- [ ] Git 提交：代码有改动？立刻 commit
- [ ] 项目说明.md：进度变了？立刻刷新
- [ ] 记忆系统：有值得存的事实？立刻 remember

## 质量把关
- [ ] 各角色的产出文件是否符合协议（去日期/四要素）
- [ ] 任务分配是否引用了正确的文件路径
- [ ] 角色对话中是否有跑偏/误读任务

## 自我纪律
- [ ] 不自己改代码——发现问题→写任务→派角色
- [ ] 外部评测/OpenClaw 报告→先验证再采信
- [ ] 任务指令精确（颜色值/尺寸/动画参数），不写"改好看点"
