# 墨境集团 · 会话安全协议

## 为什么需要
Reasonix 对话窗口是集团的"活记忆"。崩了就没了。

## 兜底方案
### 一、自动层（Reasonix 自带）
- 会话 `.jsonl` 文件自动保存
- 项目内存跨会话保留

### 二、手动层（每次收工必做）
总管必须：
1. 更新 `会话记录-日期.md`（议题+决策+产出）
2. `git commit` 代码+文档
3. 关键决策写入 `context/decisions.jsonl`
4. 全量备份到 `D:/建网站/backups/`

### 三、兜底层（如果窗口真崩了）
1. 读 `会话记录-最近日期.md` 了解进度
2. 读 `dispatch.py progress` 看各角色状态
3. 读 `context/` 恢复上下文记忆
4. 读 `集群治理协议.md` 恢复规则
5. 新对话窗口 → 输入 `/multi-agent-orchestration` → 继续说

### 四、极端情况
如果 Reasonix 整个崩了：
- Git 仓库在 D 盘 → 代码和文档全在
- 用 OpenClaw 或新 Reasonix → 加载技能包 → 继续
