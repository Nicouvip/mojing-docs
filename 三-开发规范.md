# 三、全局开发规范

## 1. TypeScript 规范

### 严格度
- 当前 `tsconfig.json` 为默认严格度（`strict: true`），但代码中频繁使用 `any` 绕过
- **目标规范**：新增代码尽量不使用 `any`，用 interface 或 type 替代
- 类型定义集中在 `src/lib/types.ts`

### 命名风格
| 类型 | 风格 | 示例 |
|------|------|------|
| 文件名 | kebab-case | `writing-editor.tsx`、`page.tsx` |
| React 组件 | PascalCase | `EditorPage`、`WritingEditor`、`Card` |
| 普通函数 | camelCase | `getProjects`、`checkCompliance` |
| 事件处理 | `handle + 名词` | `handleSave`、`handleBrainstorm` |
| 状态变量 | camelCase + 前缀 | `showReport`、`aiLoading`、`bsGenre` |
| 常量 | UPPER_SNAKE_CASE | `B_CLASS_WORDS`、`API_KEY` |
| 工具函数 | camelCase + 动词 | `calcBodyDensity`、`splitParagraphs` |

### 文件结构
```
src/
├── app/             ← 页面 + API 路由
│   ├── page.tsx     ← 路由页面
│   ├── layout.tsx   ← 根布局
│   └── api/         ← API 路由
├── components/      ← 可复用 UI 组件
│   ├── ui/          ← shadcn/ui 基础组件
│   └── *.tsx        ← 业务组件
└── lib/             ← 工具函数 + 类型 + 存储
    ├── types.ts     ← 全局类型定义
    ├── utils.ts     ← 通用工具
    ├── store.ts     ← 数据存储
    └── *.ts         ← 业务逻辑模块
```

## 2. UI 样式规范

### 色系
- 使用 Tailwind v4 的 `@theme inline` 定义 CSS 变量
- 语义化类名：`bg-background`、`text-foreground`、`text-muted-foreground`、`bg-primary`、`text-primary`
- 不直接写色值（如 `#0071E3`），始终通过变量引用

### 布局
- 全屏页面：`h-screen w-screen flex flex-col`
- 三栏布局：`flex-1 flex overflow-hidden`
- 侧栏宽度：左 220px，右面板 260px，图标栏 40px
- 编辑器排版：18px 宋体，行高 2，首行缩进 2em

### 组件
- 使用 shadcn/ui 组件库（Button、Card）
- 自定义组件放在 `src/components/` 目录
- `cn()` 工具函数处理 Tailwind 类名合并
- 弹窗用固定定位 `fixed inset-0` + `backdrop-blur` 实现

### 主题
- `globals.css` 中定义 `@theme inline` 变量
- 暗夜模式通过 `.dark` 类切换
- 暖光/冷光通过 `.theme-warm` / `.theme-cool` 类

## 3. API 规范

### 路由结构
- 路径：`/api/ai/[action]/route.ts`
- 方法：全部 `POST`
- 请求头：`Content-Type: application/json`
- 请求体：简洁 JSON 对象

### 响应格式
```typescript
// 成功
{ text: string }  // 或 { ideas: string }
// 失败
{ error: string }
// 状态码
200 — 成功
400 — 参数错误
500 — 服务器错误
```

### 错误处理
```typescript
try {
  // API 调用
} catch (e: any) {
  return NextResponse.json({ error: e.message }, { status: 500 })
}
```

### AI 模型调用
```typescript
// DeepSeek Flash（默认模型）
model: 'deepseek-v4-flash'
temperature: 0.6 ~ 1.1  // 润色用低温，创意用高温
max_tokens: 1024 ~ 2048

// Prompt 结构
"你是专业网文[角色]。[具体任务]。\n\n[上下文/输入]\n\n[输出约束]"
```

## 4. 状态和保存规范

### 编辑器状态
- 内容通过 `content` prop 下传，`onChange` 回调上传
- 使用 `useRef` 避开闭包陷阱（用于定时器等场景）
- 关键变化时手动保存（按钮）+ 30s 轮询自动保存

### 数据持久化
- 读取：`function loadClient(key, fallback)`
- 写入：`function saveClient(key, data)`
- 内存 + localStorage 双写（`setProjectsAll` / `setChaptersAll`）
- 章节选中记录：`localStorage.setItem('mojing_last_chapter_' + projectId, chapterId)`
