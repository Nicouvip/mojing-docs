# 墨境站内功能 → 成熟方案替代报告

---

## 1. store.ts localStorage → Supabase

**自建现状**：`src/lib/store.ts` 把所有数据（作品、章节、用户设置）存浏览器 localStorage。浏览器缓存清空 → 数据全丢。不同电脑看不了同一份数据。没有备份。

**Supabase 是什么**：开源 Firebase 替代品。给你一个 PostgreSQL 云端数据库 + 自动 REST API + 实时订阅 + 文件存储。
**换后效果**：数据存云端，不会丢。多设备自动同步。有权限控制。内置备份恢复。
**为什么用它**：JS 客户端用法和 localStorage 差不多，`supabase.from('projects').select()` 一行替换 `getProjects()`。

---

## 2. mock 登录 → NextAuth.js

**自建现状**：没有真实的用户系统。login 页面填什么都让进，存一个假 token 到 localStorage。没有注册、没有密码、没有安全防护。

**NextAuth.js 是什么**：Next.js 官方认证库。一行配置支持 Google / 微信 / GitHub / 邮箱密码登录。自带 JWT 会话管理、CSRF 防护。
**换后效果**：真正的用户注册登录。微信扫码登录。密码找回。会话不过期。
**为什么用它**：Next.js 官方搭档，一个文件 `src/auth.ts` 配置完成。不依赖外部服务。

---

## 3. registry.ts + ab-test.ts + feedback.ts → PromptLayer

**自建现状**：
- `registry.ts` — 手写的模板版本管理系统，代码级操作（register/activate/deactivate）
- `ab-test.ts` — 手写的 A/B 实验框架，手动分流、人工统计
- `feedback.ts` — 手写的采纳率/满意度追踪，数据只存 localStorage

**PromptLayer 是什么**：Prompt 全生命周期管理平台。提供可视化的 Prompt CMS（非技术人员也能改模板）、一行配置的 A/B 测试、自动统计的评估仪表盘。
**换后效果**：产品经理自己改 Prompt 不需要找提示词专家。A/B 对比结果自动出图表。哪个 Prompt 效果好一目了然。
**为什么用它**：不改我们的规则逻辑——只在 API 调用前后加记录层。我们管规则，它管质量。

---

## 4. iframe+PageRenderer 编辑器 → GrapesJS

**自建现状**：手写了 iframe+postMessage 方案（点击不触发、CSS 冲突、拖拽被覆盖）→ 又切 PageRenderer 方案（只渲染 3 个组件不是完整页面）→ 折腾了四轮方案都没做到可用。

**GrapesJS 是什么**：开源页面搭建器。内建拖拽组件库、样式编辑面板、图层管理器、设备预览切换、撤销/重做、JSON 保存/加载。
**换后效果**：前端已装好。拖组件到画布→改颜色/字号/间距→图层管理→手机/平板/桌面预览→保存。所有功能开箱即用。
**为什么用它**：我们手写了四轮没做出来，它一行 `grapesjs.init()` 全搞定。

---

## 5. 无 API 成本追踪 → Helicone

**自建现状**：调 DeepSeek API 不知道每次花多少钱、哪些调用最贵、有没有浪费。只能月末看账单总金额。

**Helicone 是什么**：AI API 代理网关。把 API URL 从 `api.deepseek.com` 改成 `gateway.helicone.ai`，自动记录每次调用的 token 数、延迟、成本。
**换后效果**：一眼看到哪次续写最贵、哪个 Prompt 最耗 token。精准优化成本。
**为什么用它**：改一行 baseURL，不改任何代码。

---

## 6. 手写 Toast → Sonner

**自建现状**：编辑器顶部一个 saveStatus 状态——"已保存"/"保存中"/"未保存"。只有文字，一闪而过。

**Sonner 是什么**：shadcn/ui 生态的 toast 通知库。`toast('已保存')` 一行代码弹出一个带图标的提示卡片，自动消失。
**换后效果**：保存成功 → 绿色 toast 弹出。删除作品 → 红色警告 toast。AI 续写完成 → 蓝色通知 toast。
**为什么用它**：一行代码替换手写的 saveStatus 状态机。shadcn 生态，零配置。

---

## 7. content.length 字数→ reading-time

**自建现状**：`content.length` 统计字数——HTML 标签也算进去了，字数虚高 3 倍。BUG-04 至今未彻底修复。

**reading-time 是什么**：中英文阅读时间估算库。`readingTime(text).words` 返回分词后的真实字数。
**换后效果**：中文"你好世界"统计为 4 字而不是 80 字（含 HTML 标签）。字数准确。
**为什么用它**：一行代码替换 `content.length`。彻底修 BUG-04。

---

## 8. 图片没优化 → Sharp

**自建现状**：Next.js Image 组件已在用，但没装 Sharp。图片是原始大小，不压缩、不转 WebP、加载慢。

**Sharp 是什么**：高性能图片处理库。Next.js Image 检测到 Sharp 后自动启用压缩/WebP 转换/懒加载/响应式尺寸。
**换后效果**：Logo 从 2MB 自动压缩到 200KB。首页加载快 3 倍。
**为什么用它**：装完自动生效——pnpm add sharp 后 Next.js 自动调用。零配置。
**成本**：开源免费 | **迁移**：一行安装

---

## 9. Admin 没图表 → Recharts

**自建现状**：Admin 数据看板只有数字卡片（"12 部作品""36 万字"），没有可视化图表。看不到字数增长趋势、AI 调用频率、用户活跃度。

**Recharts 是什么**：React 图表库，shadcn/ui 推荐搭配。折线图/柱状图/饼图/面积图一行组件即可。
**换后效果**：Admin 看板有"每周创作字数折线图""AI 调用趋势柱状图""题材分布饼图"。
**为什么用它**：React 原生，和 shadcn/ui 风格统一。一行 import。
**成本**：开源免费 | **迁移**：半天

---

## 10. 站内没搜索 → Pagefind

**自建现状**：没有任何搜索功能。想找某个作品只能肉眼扫列表。

**Pagefind 是什么**：静态搜索库。构建时自动索引所有页面文字，生成搜索框。零后端依赖。
**换后效果**：搜索"荒年客栈"→ 秒出结果。搜"身体密度"→ 找到合规规则页面。
**成本**：开源免费 | **迁移**：半天

---

## 11. 没 SEO → next-sitemap + Metadata API

**自建现状**：所有页面 title 都是"墨境 - 沉浸式小说写作"。没有 sitemap，Google 搜不到。

**next-sitemap 是什么**：自动生成 sitemap.xml。配合 Next.js Metadata API——每个页面有独立标题/描述/OG 标签。
**换后效果**：Google 搜"AI 网文写作工具"能搜到墨境。分享到微信有标题+缩略图。
**成本**：开源免费 | **迁移**：半天

---

## 12. 编辑器没脑图 → Mermaid.js

**自建现状**：写作前想理大纲只能新建空白文档手打。

**Mermaid.js 是什么**：Markdown 转脑图的库。用 Markdown 语法写大纲→自动渲染成思维导图。
**换后效果**：写"# 第一章
## 场景1
### 角色A出场"→自动生成大纲树。拖拽调整结构。
**成本**：开源免费 | **迁移**：半天

## 13. 没 MD 导出 → Turndown

**自建现状**：编辑器内容是 HTML（TipTap 格式），无法导出为 Markdown。想发到其他地方只能手动复制粘贴。

**Turndown 是什么**：HTML 转 Markdown 库。 一行转换。
**换后效果**：编辑器内容→一键导出 Markdown。保留加粗/斜体/标题层级。
**成本**：开源免费 | **迁移**：一行代码

---

## 14. 没 PDF 导出 → jsPDF

**自建现状**：只有手写的 TXT 导出 API。没有格式、没有排版。

**jsPDF 是什么**：前端 PDF 生成库。可控制字体/字号/页边距/分页。
**换后效果**：全本小说导出为排版精美的 PDF。带封面+目录+页码。
**成本**：开源免费 | **迁移**：半天

---

## 15. 没版本对比 → diff-match-patch

**自建现状**：改完内容不知道改了什么。想对比上一版只能靠记忆。

**diff-match-patch 是什么**：Google 开源的文本差异对比库。输入旧文本+新文本→输出差异高亮。
**换后效果**：版本历史里点"对比"→绿色=新增、红色=删除。一眼看到改了什么。
**成本**：开源免费 | **迁移**：半天

---

## 16. 自动保存太频繁 → use-debounce

**自建现状**：手写 setInterval 每 10 秒保存一次。打字中途也保存——浪费写入次数。

**use-debounce 是什么**：React 防抖 Hook。——停止打字 2 秒后才保存。
**换后效果**：不再每 10 秒写一次。打字中不保存，停下来才保存。减少 80% 写入。
**成本**：开源免费 | **迁移**：一行替换

---

## 17. 没错误监控 → Sentry

**自建现状**：500 错误只在浏览器控制台看到。不知道用户那边崩没崩、崩了多少次。

**Sentry 是什么**：前端错误监控平台。自动捕获所有 JS 异常→上报→精准定位哪一行崩了。
**换后效果**：任何用户遇到 500 → Sentry 秒级通知。带着完整堆栈信息。
**成本**：免费层 5000 事件/月 | **迁移**：半天

---

## 18. 没访问统计 → Plausible

**自建现状**：不知道有没有人在用。不知道哪个页面被访问最多。

**Plausible 是什么**：轻量隐私友好的访问统计。自部署（一个 Docker 容器）。不用 Google Analytics。
**换后效果**：首页 UV、编辑器使用时长、Admin 后台点击量。全隐私合规。
**成本**：开源免费（自部署） | **迁移**：半天

---

## 19. 没文件上传 → Uploadthing

**自建现状**：封面图/素材图靠手动复制到 public/ 目录。没有上传界面。

**Uploadthing 是什么**：文件上传服务。一行组件实现拖拽上传+进度条+图片预览。
**换后效果**：Admin 后台上传封面→自动存云端→返回 URL。用户也可以上传头像。
**成本**：免费层 2GB | **迁移**：半天

**自建现状**：没有任何邮件功能。

**Resend 是什么**：开发者邮件 API。一行代码发送验证邮件/密码重置/欢迎邮件。
**换后效果**：注册后收到"欢迎来到墨境"。忘记密码→邮箱重置。
**成本**：免费 100 封/天 | **迁移**：半天

## 21. 没支付 → LemonSqueezy

**自建现状**：没有任何付费功能。

**LemonSqueezy 是什么**：面向独立开发者的支付平台。支持支付宝/微信/信用卡。比 Stripe 接入更简单。
**换后效果**：会员三档套餐。用户扫码支付→自动升级会员等级。

---

## 22. 没离线 → PWA

**自建现状**：断网完全用不了。Next.js dev server 一停就白屏。

**PWA 是什么**：Progressive Web App。Service Worker 缓存页面→断网也能打开→联网自动同步。
**换后效果**：地铁里没信号也能打开编辑器接着写。联网后自动保存。
**成本**：Next.js 内建支持 | **迁移**：1天
